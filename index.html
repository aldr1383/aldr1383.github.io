NEUROFORCE


Client
Neurophysiological Imaging and Modeling Laboratory (NIML)

Background
Delivery of drugs to the spinal cord through the cerebrospinal fluid can provide a direct and effective route for management of pain in patients with chronic neurological disorders. However, design and optimization of drug delivery technology (e.g. infusion pumps) is difficult since in vivo studies are expensive to conduct on large animals that have similar size as humans.

Objective
To create a compact, low cost, oscillatory-flow pump which could produce a realistic cerebrospinal fluid (CSF) flow conditions within a given range of human/animal cerebrospinal to be connected to a physical model of subarachnoid space.

Role
Design and prototype internal mechanical system and overall assembly.

Design specifications
Compact, Fit in your pocket, Adjustable Volume, MRI-compatible, Easy to use, 5 V trigger, Low cost, Consistent (Reliability +- 5%), Bleed system for piping, Adjustable Frequency (30-230 BPM), Simple interface.

Approach
- For designing the assembly, I used Solidworks for its motion and animation capability.
-To create our initial prototype, we used a 3d printer (Ultimaker 2+) for rapid prototyping due to time constraint.
- Different mechanical systems were explored to create the oscillatory flow that we wanted while still produce a product that is within the size constraint.
- Different 3d printing materials were tested to create a durable product.
- Use a controller pre-built controller hardware.
- Use Fourier Series to obtain function.

Result
A complete working model was developed in Solidworks CAD software, all specifications were met except for MRI- compatibility. Our product ended up costing around $500. Initially, we decided that our final product would be built using a 3d printer but decided later to purchase pre-built case for the outer shell. We made this decision for a number of reasons; time constraint; cost; weight. The whole assembly of the first prototype and 1 part on the final product (a connector to the syringe and the linear actuator) was created using Ultimaker 2+. Polycarbonate was chosen for the material due to its strength and durability. Scotch and Yoke mechanism connected to a dc motor was originally chosen but we eventually chose to purchase a linear actuator. The linear actuator was chosen due to its size and a significantly better wave output. We use Arduino DUE for the microcontroller board. Fourier series analysis was used on raw data provided by NIML to obtain the function that mimics the Cerebrospinal fluid flow. A potentiometer and a pressure transducer were used to compare to compare reading of the position of the linear actuator and the wave produced inside the spinal column.

Subject Specific Physiologic Simulator of Intracranial Fluid Dynamics


Client
Neurophysiological Imaging and Modeling Laboratory (NIML)

Background
The ongoing research at the Neurophysiological Imaging and Modeling Laboratory models CSF system by computer (in silico), physical model (in vitro) and virtual reality. A detailed in vitro model would allow for testing of medical devices. Many companies are developing potentially therapies for treatment of CNS disorders such as Alzheimer’s Amyotrophic Lateral Sclerosis (ALS), Parkinson’s and autism. These therapies for brain disorders require extremely expensive and ethically charged non-human primate studies to reach clinical use. Such expenses make detailed testing and optimization of brain therapeutic systems and medications difficult. To do that, a cost-effective system that accurately represents the CSF anatomy and flow is needed.

Introduction
The cerebrospinal Fluid (CSF) is a clear water-like fluid that bathes the brain and the spinal cord and also resides in a series of complex interconnected spaces within the brain called ventricles and cisterns. Understanding the CSF system is crucial for diagnostic of central nervous system (CNS) diseases. CNS disease can be difficult to treat because of the blood brain barrier (BBB) which prevent drugs that could potentially help CNS disorder from entering the brain. One way to bypass BBB is through injection of drugs to CSF. Therefore, cerebrospinal fluid is potentially and important route to deliver drugs to the CNS.

Method
Ultra-high resolution (300 um isotropic) MRI -> Anatomical Review -> segmentation in ITK-SNAP -> Processing in Blender -> Stereolithography 3-D printing
1 .The 3D model of the cortical subarachnoid space, ventricles, and cisterns of the brain was created using Ultra-high resolution (300 um isotropic) MRI scans (A). The 3D model was segmented using a medical image processing software called ITK-SNAP with consistent segmentation process leaving 2 pixels unsegmented to prevent over segmentation (B). The location and dimension of the intracranial CSF system was determined from a variety of neurological peer-reviewed papers. The model was then exported as standard tessellation language files and imported into an open source modelling program.
2. The geometry will then be smoothened to make it more anatomically realistic in Blender. Watertight connections will be added to the model that will be used to attach tubing to the system. The model will then be split into pieces that can be readily 3D printed and assembled. The segmented model will be printed using stereolithography (SLA) 3-D printing to create a high-resolution physical model of the intracranial system.

Preliminary result geometry
A 3D model of the intracranial cerebrospinal fluid system including the cortical space created from an Ultra-high-resolution (300 um isotropic) MRI scans. The images were segmented in ITK-SNAP with a consistent segmentation process.

Future work
1. Process current 3D geometry:
- Smooth current 3D geometry using blender making it more anatomically realistic.
- Add connection to the model for tubing system. Split 3d model for printing.
2. 3D printing:
- Print 3D model using stereolithography printing.
- Explore the best material that would give the best result.
3. Preliminary testing.
- Conduct basic pressure test.
- Check overall volume and instrument model with a pressure transducer, unsteady pressure will be recorded as a function of delivered flow waveform.

Here is the quotechanger constant which you can find in the js script.

let m = Math.floor(Math.random()*quote.length) const quoteChanger = (state = m, action) => {

Markdown Previewer

See the Pen Mark Down Previewer by aldr1383 (@aldr1383) on CodePen.


Languages
Javascript, React, Redux, CSS,HTML.

Project goals
- Create an interface for user to input markdown text and render it as HTML.
- When the markdown previewer first loads, the default text in the editor field should contain valid markdown that represents at least one of the each of the following elements: a header (H1 size), a sub HEADER (h2 SIZE), a link, inline code, a code block, a list item, a blockquote, an image, and bolded text.

Background
Markdown is a plain text formatting syntax aimed at making writing for the internet easier. By using simple syntax, different text modifiers like lists, bold and italics can be added.

An example of of markdown syntax is *bold word*. When this syntax is used, it will look like bold word.

To learn more about the syntax you can click here

Solving the problem
I used react-redux to create the interface and state management.

I created a variable called initial to store the initial markdown text containing all of the requirements when the page first load. When a change is detected in the text area, an action is dispatched and record the value. The value is then processed using the marked() function.

I imported a marked library from cdnjs. Initial state is set to equal the variable initial. At default state, the function marked is used on this initial statereturn marked(state) and upon change marked() is used on the new state, which is the text in the textarea with any modifications made to it, return marked action message.

Drum Machine

See the Pen Build a drum machine by aldr1383 (@aldr1383) on CodePen.


Project goals
- Create 9 clickable drump pad elements, each with a class name of drum-pad, a unique id that describes the audio clip the drum pad will be set up to trigger, and an inner text that corresponds to one of the following keys on the keyboard: Q, W, E, A, S, D, Z, X, C.

- Within each .drum-pad, there shoud be an HTML5 audio element which has a src attribute pointing to am audio clip, a class name of clip, and an id corresponding to the inner text of its parent .drum-pad (eq id="Q", id="W", id="E" etc.).

- When the .drum-pad element is clicked, the audio clip contained in its child audio element should be triggered.

- When the trigger key associated with each .drum-pad, the audio clip contained in its child audio element should be triggered (e.g. pressing the Q key should trigger the drum pad which contains the string "Q", pressing the W key should trigger the drum pad which contains the string "W", etc.).

- When a .drum-pad is triggered, a string describing the associated audio clip is displayed as the inner text of the #display element (each string must be unique).

Solving the problem
Since the string inside the button matches the id of the audio child element, I was able to target the audio element and used .play() to play the audio using the following code

document.getElementById(event.currentTarget.textContent)

Adding .play() and inserting this code into handleClick() allows the user to play the audio element associated with the button. The description of the audio clip is in the id of the button element allowing me to target this id and return it as a new state in Redux. In the following line of codes, the id(description) is set to a variable called kop and passed as the new state which is then displayed below the buttons.

handleClick(event) {
let kop= event.currentTarget.id
document.getElementById(event.currentTarget.textContent).play()
this.props.submitNewMessage(kop)
}
